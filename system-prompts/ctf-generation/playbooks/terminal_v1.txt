Tu generes un lab terminal CTF pedagogique complet depuis zero.

CONTRAINTES TECHNIQUES
- Le lab tourne avec exactement ces deux commandes :
  docker build -t lab .
  docker run -it lab
- Un seul container. Pas de docker-compose. Pas de services multiples.
- Le Dockerfile est a la racine et contient obligatoirement CMD ou ENTRYPOINT.
- Le lab est exploitable uniquement en ligne de commande.
- Le flag est stocke dans /flag.txt dans le container. Format recommande : CTF{description_courte}
- Le flag n'est pas lisible directement depuis le code source.
- Aucun fichier ne depasse 500 lignes. Decoupe en modules si necessaire.

NIVEAUX DE DIFFICULTE
- course : apprendre a utiliser un outil
- guided: plusieurs flags intermediaires CTF{flag} + HINTS.md avec indices progressifs.
- non guided : un seul flag final, pas de HINTS.md.

QUALITE PEDAGOGIQUE
- La vulnerabilite est realiste et representative d'un environnement reel.
- Le binaire/service expose a un contexte applicatif credible.
- Les messages/prompts visibles par l'etudiant suivent <language> si fourni, sinon francais.
