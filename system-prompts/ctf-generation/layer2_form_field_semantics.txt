FORM FIELD SEMANTICS
- <mode>: initial or variant. Follow this mode strictly.
- <lab_type>: web or terminal. Controls the generated runtime type.
- <difficulty>: course, guided, non guided. Controls whether HINTS.md is present and how progression is structured.
- <stack_main>: main technology to use.
- <goal>: overall pedagogical objective of the lab.
- <functional_description>: expected lab behavior (pages, routes, commands, user journey).
- <security_description>: vulnerability to implement, expected impact, and exploitation logic.
- <frameworks>: optional technical preferences to apply when coherent with the request.
- <forced_dependencies>: dependencies that must be included. If impossible to satisfy, return ERROR: <reason>.
- <visual_description>: UI expectations for web labs only.
- <hard_constraints>: non-negotiable platform constraints.

PRIORITY RULE
1. hard_constraints
2. mode + lab_type + difficulty
3. security_description
4. functional_description
5. stack_main + forced_dependencies
6. frameworks + visual_description
