OUTPUT CONTRACT - REQUIRED

Each file must be produced using exactly this format:

<<<FILE: path/to/file.ext>>>
[file content]
<<<END>>>

Absolute rules:
- Start immediately with the first <<<FILE:>>> block.
- No text before the first <<<FILE:>>> block.
- No text after the final <<<END>>>.
- No markdown fences (```) inside or around file blocks.
- Each file has its own <<<FILE:>>> / <<<END>>> block. Never put two files in one block.
- The path in <<<FILE: path>>> is relative to the project root.

HARD CONSTRAINTS :
1. Do not finish your response until all files required for the lab to run have been output, including the mandatory Dockerfile.
2. Every file referenced by another file (import, require, COPY in Dockerfile) must exist in your output.
3. The final flag must use the CTF{...} format and must not be directly readable from the source code. In guided labs, intermediate CTF{step_N_...} flags are allowed.
4. If the difficulty is guided, do not finish without producing HINTS.md. If the difficulty is non guided, HINTS.md is forbidden.
5. Any file whose content exceeds 500 lines is an error. Split it into modules before outputting it.

ANTI-PATTERNS :
1. Never produce free text between two <<<FILE:>>> / <<<END>>> blocks.
2. Never wrap file content in markdown fences.
3. Never produce a <<<FILE:>>> block without its matching <<<END>>> on a line by itself.
4. Never split one file into multiple <<<FILE:>>> blocks with the same path.
5. Never produce docker-compose.yml, docker-compose.yaml, or any Compose file. The lab is single-container only.
